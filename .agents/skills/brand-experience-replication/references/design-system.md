# Design System Profile (Canonical Reference: Sadhaka)

## Purpose

Translate brand intent into implementable visual and interaction rules.

## Design principles

1. Premium calm over visual aggression
2. High readability in dark-theme dominant surfaces
3. Controlled depth (glass + gradient + shadow) without clutter
4. Consistent spacing rhythm and rounded geometry

## Token categories to extract or define

### Color roles (semantic)
- `bg.base` — primary page background
- `bg.elevated` — cards, overlays
- `surface.glass` — translucent premium surfaces
- `text.primary` / `text.secondary`
- `brand.primary` / `brand.accent`
- `state.success` / `state.warning` / `state.error`
- `border.subtle` / `border.strong`

### Typography
- Display scale for hero promise
- Heading tiers for section narrative
- Body styles for readable long-form explanation
- Microcopy/caption style for trust chips and metadata

### Spatial system
- Section vertical rhythm (large, medium, compact)
- Component internal spacing (sm/md/lg)
- Grid behavior across breakpoints

### Shape and depth
- Radius scale for cards/buttons/chips
- Shadow/elevation tiers
- Blur and transparency settings for glass surfaces

### Motion and interaction
- Hover/press/focus states
- CTA emphasis transitions (subtle, not flashy)
- Scroll reveal behavior with restraint

## Component behavior patterns

### Hero
- clear value promise
- one primary CTA + one secondary CTA
- immediate trust cues near CTA cluster

### Header/navigation
- concise nav choices
- persistent primary CTA visibility
- avoid menu overload

### Trust chips / micro-proof
- short, scannable evidence claims
- icon + text consistency

### Educational/pillar blocks
- explain mechanism, not just outcome
- sequence from concept clarity to practical value

### Social proof module
- use verifiable proof formats (testimonials, logos, quantified outcomes)
- avoid ungrounded praise-only statements

### Conversion section
- restate value in outcome language
- handle final objections before ask
- maintain single conversion intent

## Art direction

- Abstract but meaningful visuals (not random decoration)
- Domain-relevant metaphors when adaptation requires them
- Keep visual language cohesive with typography and card style

## Accessibility and quality floor

- maintain contrast for dark surfaces
- readable line-length and font sizing
- visible keyboard focus states
- avoid motion that harms readability or comfort
