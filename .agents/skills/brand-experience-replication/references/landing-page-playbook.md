# Landing Page Playbook (Canonical Pattern)

## Objective

Provide a reusable section sequence and conversion logic that reproduces the Sadhaka-style experience without literal cloning.

## Canonical section order

1. **Hero**
2. **Immediate trust cues**
3. **Problem reframing / stakes**
4. **Pillars or mechanism explanation**
5. **Social proof and credibility layer**
6. **How it works / process clarity**
7. **Offer framing / plan fit**
8. **Objection handling + FAQ**
9. **Final conversion section**

## Section intent and implementation notes

### 1) Hero
- Goal: establish positioning and emotional promise quickly.
- Components: headline, subhead, primary CTA, secondary CTA, trust chips.
- Rule: primary CTA must match page-level conversion intent.

### 2) Immediate trust cues
- Goal: reduce first-scroll skepticism.
- Components: concise proof chips, logos, credentials, security/compliance indicators.

### 3) Problem reframing / stakes
- Goal: help users feel seen and clarify cost of inaction.
- Components: pain articulation + consequence framing.

### 4) Pillars/mechanism
- Goal: show *why* this works, not just that it exists.
- Components: 3–5 principle cards with short explanatory copy.

### 5) Social proof
- Goal: transfer trust from peers/market evidence.
- Components: testimonials, outcomes, logos, case snippets.
- Rule: keep claims verifiable and specific.

### 6) Process clarity
- Goal: lower effort anxiety by showing journey.
- Components: step-by-step flow, timeline, expected outputs.

### 7) Offer framing
- Goal: align product/package to user context.
- Components: plan cards or package framing with clear boundaries.

### 8) Objection handling + FAQ
- Goal: answer pre-purchase concerns directly.
- Components: strongest objections first, plain-language answers.

### 9) Final conversion section
- Goal: summarize value and drive action.
- Components: concise value recap + single primary CTA + low-friction reassurance.

## CTA logic

- Exactly one primary CTA intent per major section.
- Secondary CTA can support learning path (e.g., “See how it works”).
- Don’t switch CTA destination unpredictably across sections.

## Objection handling framework

Address these in sequence:
1. Fit (“Is this for someone like me?”)
2. Credibility (“Will this actually work?”)
3. Effort (“How hard is implementation?”)
4. Risk (“What if this fails?”)
5. Timing (“Why act now?”)

## Quality checks

- Can a first-time visitor explain the offer in 10 seconds?
- Is proof introduced before major asks?
- Is each section doing one job clearly?
- Are there any conflicting CTA messages?
