# Extraction Checklist — Tradebooks AI Brand System

Use this checklist before generating brand, design, or conversion outputs from the skill to ensure all necessary context has been captured.

## 1) Brand strategy extraction

- [ ] Mission and problem context captured (Zerodha tradebook → Tally XML pipeline)
- [ ] Primary audience segment defined (trader / CA / prop desk)
- [ ] Emotional promise identified (certainty over the data, not just time saved)
- [ ] Trust posture and conversion philosophy documented (mechanism clarity before ask)
- [ ] Tone calibrated to segment (trader = direct, CA = professional peer)

## 2) Design system extraction

- [ ] Semantic color roles confirmed (navy base, teal primary, green/amber/red/blue states)
- [ ] Typography family confirmed (Inter for all; monospace for data/file references)
- [ ] Spacing and radius scale appropriate for surface (landing = generous; app = compact)
- [ ] Motion rules checked (purposeful only — no decorative float/pulse)
- [ ] State chips verified: always icon + label + color (never color alone)

## 3) Landing architecture extraction

- [ ] Section order mapped (problem → mechanism → proof → offer → CTA → FAQ)
- [ ] Hero structure documented (problem-named headline + output-specified subhead + direct CTA)
- [ ] Pipeline steps confirmed (Upload / Process / Export — user-effort framed)
- [ ] Proof modules verified (specific outcome testimonials + volume signals)
- [ ] Final conversion framing confirmed (concise recap + single CTA + reassurance)

## 4) Component pattern extraction

- [ ] Trust chip rail captured (specific, verifiable claims — not generic badges)
- [ ] Pipeline step pattern captured (3 steps, headline + one-sentence note)
- [ ] Feature pillar pattern captured (mechanism explanation, not feature-list bullets)
- [ ] Social proof pattern captured (outcome-specific quotes, not generic praise)
- [ ] FAQ pattern captured (6 key objection clusters: fit, accuracy, compatibility, effort, privacy, timing)
- [ ] Batch dashboard states captured (green/amber/red/blue status chips)

## 5) Adaptation readiness

- [ ] Audience segment explicitly specified for this execution
- [ ] Invariants locked (accuracy-first, domain-native language, single CTA, proof-before-ask)
- [ ] Variables specified (CTA label, proof type, feature emphasis, broker specificity)
- [ ] Guardrail check run: no vague AI claims, no overpromised tax outcomes
- [ ] Domain-specific trust translation confirmed: broker/Tally compatibility signals front-loaded
